Découvertes & Surprises

✅

AIMultiple Confirmé

Nos résultats confirment ceux d'AIMultiple : GPT-5 domine l'écriture cursive, Gemini 2.5 Pro est le plus fiable, et Claude Sonnet 4.5 est bon sans être exceptionnel.

🔍

Révélation : Qwen3.5 122B

Absent du benchmark original, ce modèle GRATUIT est le seul à avoir transcrit intégralement les deux questions et le diagramme du formulaire BTS. Rapport qualité/prix imbattable.

❌

GPT-5 Bloqué par le Filtre

GPT-5 a totalement refusé de transcrire le Document 1 — réponse vide. Probablement le filtre de sécurité d'OpenAI déclenché par le contenu d'examen technique (code + schémas).

🤖

Claude : Fort à l'Écran, Faible sur Papier

Claude excelle sur la cursive continue mais donne une sortie minimaliste sur les documents structurés. Son système de vision semble optimisé pour les captures d'écran, pas les formulaires scannés.

📄

GLM OCR : Parfait pour l'Imprimé, Nul pour la Cursive

GLM OCR détecte parfaitement la mise en page (tableaux HTML, bounding boxes) mais détruit complètement l'écriture cursive. À utiliser comme prétraitement pour les documents imprimés avant transcription par LLM.

Modèles Non Testés

4 modèles du benchmark AIMultiple n'ont pas pu être testés — ils nécessitent des instances GPU dédiées ou des clés API propriétaires

🔬

olmOCR-2-7B-1025-FP8

Top 4 du benchmark

Modèle OCR spécialisé d'Allen AI. Non disponible via API serverless. Nécessite un HF Inference Endpoint (~1-3 $/h de GPU) ou une installation locale avec conda + vLLM.

👁️

Moondream OCR

Milieu de tableau

Modèle de vision avec OCR. Non disponible via API serverless. Nécessite une clé API moondream.ai ou une installation locale avec GPU NVIDIA.

🤖

DeepSeek OCR

Bas de tableau

Modèle OCR centré LLM. Répertorié comme "image-text-to-text" sur HF mais non compatible avec l'API chat. Nécessite un déploiement vLLM ou un endpoint dédié.

🐼

PaddleOCR-VL

Bas de tableau

Pipeline OCR de Baidu. Non disponible sur HuggingFace. Nécessite Python 3.13+ : pip install paddleocr.

Les API cloud spécifiques (Azure Cognitive Service, Google Vision, Amazon Textract) et les produits commerciaux (Dots OCR, Nanonets, Mistral OCR) n'ont pas non plus été testés — ils nécessitent des comptes fournisseurs et une facturation séparée.
