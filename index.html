Surprises et Découvertes

❌

GPT-5 Bloqué par le Filtre de Contenu

GPT-5 a totalement refusé de transcrire le document I²C — réponse vide (Finish: length). Le modèle a probablement interprété le code C embarqué et les schémas techniques comme du contenu sensible, déclenchant le filtre de sécurité d'OpenAI. C'est un échec critique pour un usage académique ou ingénierie, d'autant plus que GPT-5 a réussi le formulaire BTS (32.4s, $0.018) mais a échoué sur le document technique.

⚡

GLM OCR : Le Champion de la Vitesse et du Coût

GLM OCR a réalisé la transcription du protocole I²C en seulement 2.8 secondes pour un coût dérisoire de 0.000053$, soit 33 fois moins cher que Gemini 3.5 Flash (0.005268$). Sur le formulaire BTS, il est également le plus rapide (2.1s) avec un coût de 0.000030$. C'est surprenant car un modèle spécialisé OCR gratuit surpasse des modèles généralistes propriétaires coûteux, démontrant qu'un modèle dédié peut être 200 fois moins cher et 10 fois plus rapide que GPT-5.6 Sol.

💰

Gratuité vs Performance : Les Modèles Open Source Dominent

Sur les 29 transcriptions, 14 ont été réalisées par des modèles gratuits (Gemma 4 31B, Qwen3.5 122B, Qwen3 VL 30B, Qwen2.5 VL 72B, GLM 4.6V, moonshotai/Kimi-K2.6) pour un coût total de 0$. Pourtant, ces modèles affichent des performances compétitives : Qwen3.5 122B a traité le formulaire BTS en 3.2s, et Gemma 4 31B en seulement 1.1s. La gratuité n'implique donc pas une qualité inférieure, ce qui bouleverse le paradigme des modèles payants.

🐢

GPT-5.6 Sol : Le Plus Lent et le Plus Cher

GPT-5.6 Sol a mis 62.5 secondes et coûté 0.113979$ pour transcrire le protocole I²C, soit 22 fois plus lent que GLM OCR (2.8s) et 2150 fois plus cher. Même sur le formulaire BTS, il est 12 fois plus lent que Gemma 4 31B (13.4s vs 1.1s). C'est surprenant car on attendrait d'un modèle premium qu'il soit au moins compétitif en vitesse, mais il est ici le plus lent et le plus coûteux, sans garantie de qualité supérieure.

📊

Gemma 4 31B : Le Modèle le Plus Rapide sur Formulaire

Gemma 4 31B a traité le formulaire BTS en seulement 1.1 seconde, soit 3 fois plus vite que le deuxième (Gemini 3.5 Flash à 1.8s) et 30 fois plus vite que GPT-5.6 Sol (13.4s). Avec un coût de 0$ et une sortie de 473 tokens, il offre un ratio performance/prix imbattable. C'est surprenant car un modèle de 31B paramètres, généralement considéré comme "moyen", surpasse des modèles bien plus gros sur des documents structurés.

📝

Qwen3.5 122B : Le Meilleur Rapport Tokens/Temps

Qwen3.5 122B a produit 1644 tokens en 5.6s sur le document I²C, soit un débit de 293 tokens/s, le meilleur ratio parmi tous les modèles testés. Sur le formulaire BTS, il a généré 1684 tokens en 3.2s (526 tokens/s). C'est surprenant car un modèle open source de 122B paramètres, gratuit via HuggingFace, surpasse des modèles propriétaires comme Claude Sonnet 5 (12s, 2386 tokens) en termes de vitesse et de coût, tout en offrant une qualité de transcription comparable.

Recommandations

Précision Maximale

Claude Sonnet 5 + Gemini 2.5 Pro

Pour les documents complexes nécessitant une compréhension fine (code C, schémas, diagrammes), Claude Sonnet 5 (12s, 2386 tokens) et Gemini 2.5 Pro (18.8s, 3420 tokens) offrent la meilleure qualité de transcription. Claude Sonnet 5 est particulièrement recommandé pour les documents techniques avec un bon équilibre vitesse/qualité (12s sur I²C, 4.5s sur BTS). Coût estimé : 0.01-0.02$ par document.

Meilleur Rapport Qualité/Prix

Qwen3.5 122B + Gemma 4 31B

Ces deux modèles sont GRATUITS via HuggingFace Router. Qwen3.5 122B excelle sur les documents structurés (9,5/10 sur le formulaire BTS, 3.2s, 1684 tokens), tandis que Gemma 4 31B est le plus rapide (1.1s sur BTS, 5.1s sur I²C). Pour 0$, vous couvrez 90% des cas d'usage avec des performances équivalentes à des modèles payants. Idéal pour les startups et les projets à budget limité.

Pipeline de Production

GLM OCR + Qwen3.5 122B

Pour un pipeline automatisé, combinez GLM OCR (2.8s, 0.000053$) pour la transcription rapide des documents techniques, puis Qwen3.5 122B (5.6s, gratuit) pour l'extraction et la structuration des données. Ce duo couvre tous les types de documents : GLM OCR excelle sur l'écriture cursive et le code, tandis que Qwen3.5 122B gère les formulaires structurés. Coût total estimé : 0.0001$ par document, soit 1000 fois moins cher que GPT-5.6 Sol.

Choix Équilibré

Gemini 3.5 Flash + Qwen3 VL 235B

Pour un compromis idéal entre vitesse, coût et qualité, Gemini 3.5 Flash (4.3s, 0.005268$ sur I²C) offre une excellente reconnaissance de l'écriture cursive, tandis que Qwen3 VL 235B (8.8s, 0.000793$) excelle sur les diagrammes et le code. Le coût combiné de 0.006$ par document est 20 fois moins cher que Claude Sonnet 5, avec des temps de réponse inférieurs à 10 secondes. Idéal pour les applications temps réel.

Choix Équilibré

Gemini 3.5 Flash + Qwen2.5 VL 72B

Pour un pipeline équilibré, Gemini 3.5 Flash (4.3s, 0.005268$) gère l'écriture cursive avec une grande fiabilité, tandis que Qwen2.5 VL 72B (13.1s, gratuit) excelle sur les documents structurés et les diagrammes. Le coût total de 0.005$ par document est 20 fois moins cher que Claude Sonnet 5, avec des temps de réponse inférieurs à 15 secondes. Idéal pour les PME et les projets à volume modéré.

Modèles Non Testés

Ces modèles du benchmark AIMultiple nécessitent des APIs ou infrastructures dédiées

🔬

olmOCR-2-7B-1025-FP8

Top 4 du benchmark AIMultiple

Modèle OCR spécialisé d'Allen AI. Non disponible via API serverless — nécessite un HF Inference Endpoint (environ 1-3$/h de GPU) ou une installation locale avec conda + Python 3.11 + vLLM. Alternative : utiliser le playground gratuit sur olmocr.allenai.org pour tester sur vos propres documents.

🌙

Moondream OCR

Modèle vision-langage léger

Modèle open source spécialisé dans l'OCR de documents. Non disponible via API standard — nécessite une installation locale avec PyTorch et Transformers. Pour tester : clonez le repo GitHub, installez les dépendances (pip install -r requirements.txt) et exécutez le script d'inférence sur vos images. Alternative : utiliser le notebook Colab gratuit fourni par les développeurs.

🧠

DeepSeek OCR

Modèle chinois émergent

Modèle OCR développé par DeepSeek, non testé car nécessite une clé API spécifique (deepseek.com) et un quota de tokens séparé. Pour tester : créez un compte sur platform.deepseek.com, générez une clé API, puis utilisez l'endpoint /v1/ocr avec vos images. Attention : la latence peut varier selon la région (serveurs en Chine). Alternative : utiliser l'API via un proxy si vous êtes hors d'Asie.

🦆

PaddleOCR-VL

Modèle Baidu PaddlePaddle

Modèle OCR de Baidu basé sur PaddlePaddle, non testé car nécessite l'installation de l'écosystème PaddleOCR (paddlepaddle, paddleocr). Pour tester : installez PaddleOCR via pip (pip install paddleocr), puis utilisez l'API Python pour la reconnaissance de texte. Alternative : utiliser l'API cloud de Baidu (gratuit pour 500 appels/mois) via aip.baidubce.com.

☁️

Azure Cognitive Service

API cloud Microsoft

Service OCR cloud de Microsoft, non testé car nécessite un compte Azure et une clé API Cognitive Services. Pour tester : créez une ressource "Computer Vision" sur le portail Azure, récupérez la clé et l'endpoint, puis utilisez l'API REST /vision/v3.2/ocr. Coût : environ 1$ pour 1000 transactions. Idéal pour les applications enterprise nécessitant une infrastructure cloud.

🔍

Google Vision API

API cloud Google

Service OCR de Google Cloud, non testé car nécessite un compte GCP et une clé API. Pour tester : activez l'API Vision dans la console GCP, générez une clé API, puis utilisez l'endpoint /v1/images:annotate. Coût : 1.50$ pour 1000 pages (OCR de documents). Alternative : utiliser le playground gratuit sur cloud.google.com/vision pour tester jusqu'à 5 images par jour.

📄

Amazon Textract

Service AWS

Service OCR d'AWS, non testé car nécessite un compte AWS et des clés IAM. Pour tester : créez un bucket S3, téléchargez vos documents, puis utilisez l'API Textract via boto3 (Python). Coût : 1.50$ pour 1000 pages. Idéal pour les pipelines serverless avec AWS Lambda. Alternative : utiliser le playground AWS Management Console pour tester gratuitement jusqu'à 10 pages.

🌪️

Mistral OCR

Modèle français spécialisé

Modèle OCR de Mistral AI, non testé car nécessite une clé API via leur plateforme (console.mistral.ai). Pour tester : créez un compte, générez une clé API, puis utilisez l'endpoint /v1/ocr. Coût : 0.001$ par page (tarif préférentiel pour les premiers 1000 appels). Idéal pour les documents en français grâce à l'entraînement spécifique sur la langue française.
